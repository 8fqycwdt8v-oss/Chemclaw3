# `docs/adr/` — Architecture Decision Records

One file per decision, numbered and append-only (`NNNN-short-title.md`). An ADR
captures the context, the decision, and its consequences so a new developer
understands *why* from the record alone (gate G5).

ADRs are the long-form companion to `DECISIONS.md` at the repo root: `DECISIONS.md`
is the terse running log (D-001, D-002, …); an ADR here expands a decision that
needs full context. Not every `DECISIONS.md` entry needs an ADR — add one when
the rationale outgrows a paragraph.
